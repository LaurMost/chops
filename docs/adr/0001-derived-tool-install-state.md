# Skill–tool install state is derived from discovery, not stored

A skill's set of installed tools (`toolSources`) is computed entirely from where its file/symlink is discovered on disk during scanning — there is no separate persisted "installed in tool X" record. We considered adding an explicit stored install set (which would require a new SwiftData schema version + migration), but rejected it: the filesystem is already the source of truth, a symlinked skill naturally collapses to one entry via `resolvedPath`, and "install to tool" / "not-yet-installed tools" are both computable as set operations over `toolSources` and the installed tools. This keeps the model thin and migration-free; the cost is that install state is only as fresh as the last scan, and every install/uninstall action must trigger a rescan to be reflected in the UI.
